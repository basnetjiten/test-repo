# Skill: web-scaffolder

## When to Use This Skill

| Condition in Task Plan | When to Invoke |
|---|---|
| Plan creates a new route or page (any route group) | Always — read Directory Structure + Page Shell sections |
| Plan adds a new feature module in `src/modules/` | Read Module Structure section |
| Plan adds a new component group under `src/components/` | Read Component Organization section |
| Plan scope is `full_feature` | Read all sections before writing any files |

## Directory Structure

All feature pages live under `src/app/{route-group}/`. Components live under `src/components/shared/`. Feature logic lives under `src/modules/`.

```
src/
├── app/
│   ├── (dashboard)/           ← route group — authenticated pages (MainLayout)
│   │   └── {feature}/
│   │       ├── page.tsx       ← entry — can be server or client component
│   │       ├── layout.tsx     ← optional: wraps sub-routes
│   │       ├── loading.tsx    ← optional: streaming suspense fallback
│   │       └── {sub-route}/
│   │           └── page.tsx
│   ├── (minimal)/             ← route group — auth pages (MinimalLayout)
│   │   └── auth/
│   │       ├── login/
│   │       │   └── page.tsx
│   │       └── register/
│   │           └── page.tsx
│   └── (simple)/              ← route group — public pages (SimpleLayout)
│       └── about/
│           └── page.tsx
├── modules/
│   └── {feature}/
│       ├── index.tsx          ← main page component
│       ├── graphql/
│       │   ├── queries.graphql
│       │   ├── mutations.graphql
│       │   ├── queries.generated.ts  (auto-generated by codegen)
│       │   └── mutations.generated.ts (auto-generated)
│       ├── hooks/
│       │   └── useGQL.tsx     ← Apollo hooks wrapper
│       └── constants/
│           └── types.ts       ← feature-specific types
├── components/
│   └── shared/
│       └── {feature}/
│           ├── {FeatureName}List.tsx
│           ├── {FeatureName}Form.tsx
│           └── index.ts
├── store/
│   └── slices/
│       └── {feature}.ts       ← Redux Toolkit slice
├── menu-items/
│   └── {feature}.tsx          ← sidebar navigation definition
└── types/
    └── {feature}.ts           ← shared TypeScript types
```

## Scaffold Commands

Create the directory skeleton before writing files:
```bash
# Route group — choose one based on auth requirements:
#   (dashboard) = authenticated, MainLayout
#   (minimal)   = auth pages, MinimalLayout
#   (simple)    = public pages, SimpleLayout
mkdir -p "src/app/(dashboard)/{feature}"
mkdir -p "src/modules/{feature}/graphql"
mkdir -p "src/modules/{feature}/hooks"
mkdir -p "src/modules/{feature}/constants"
mkdir -p "src/components/shared/{feature}"
```

## Page Shell Template

```typescript
// src/app/(dashboard)/{feature}/page.tsx
import { Metadata } from 'next';

export const metadata: Metadata = {
  title: '{Feature Name} | App',
  description: '{Feature description}',
};

// If the page needs client interactivity, import and render a client component from modules/
// import { FeatureView } from '@/modules/{feature}';
export default function FeaturePage() {
  return <div>{/* render FeatureView here */}</div>;
}
```

```typescript
// src/modules/{feature}/index.tsx — main client component for the feature
'use client';

import { Box, Typography } from '@mui/material';

const FeatureView = () => {
  return (
    <Box sx={{ p: 3 }}>
      <Typography variant="h4">Feature Name</Typography>
    </Box>
  );
};

export default FeatureView;
```

## File Naming Conventions

| Type | Convention | Example |
|---|---|---|
| Page/Route | `page.tsx` | `src/app/(dashboard)/products/page.tsx` |
| Layout | `layout.tsx` | `src/app/(dashboard)/products/layout.tsx` |
| Module page | `index.tsx` | `src/modules/products/index.tsx` |
| Component | `PascalCase.tsx` | `ProductCard.tsx` |
| Hook | `use{Name}.ts` | `useProductForm.ts` |
| Redux slice | `camelCase.ts` | `src/store/slices/product.ts` |
| GraphQL query | `camelCase.graphql` or `queries.ts` | `src/modules/products/graphql/queries.graphql` |
| Menu item | `camelCase.tsx` | `src/menu-items/products.tsx` |
| Type | `camelCase.ts` | `src/types/product.ts` |
| i18n | `en.json` (add keys to existing file) | `src/utils/locales/en.json` |

## Rules

- Route files (`page.tsx`, `layout.tsx`, `loading.tsx`, `error.tsx`) are NEVER deleted — they define the URL tree
- `error.tsx` MUST include `'use client'` directive
- Use route groups `(dashboard)`, `(minimal)`, `(simple)` to apply shared layouts — do NOT create routes outside these groups
- Authenticated features go in `(dashboard)`, auth-only pages in `(minimal)`, public pages in `(simple)`
- Always add SEO metadata (`export const metadata`) on every `page.tsx`
- Feature modules go in `src/modules/`, NOT in `src/lib/`
- Import alias `@/` maps to `src/` — use for all project imports
